##  API 概述

### 1.1 设计原则

本项目 API 遵循 RESTful 设计规范，后端采用 Java Spring Boot 框架进行开发。系统不仅提供标准的数据增删改查服务，还深度集成了大语言模型（LLM）、检索增强生成（RAG）以及知识图谱（Knowledge Graph）功能，以支持用户的个人成长与知识管理。


### 1.2 基础信息

**支持格式**：
- 支持格式：请求与响应均采用`application/json`格式
- 数据模型基类：系统的数据传输主要依赖定义好的 DTO（Data Transfer Object）与实体类（Entity）。

##  API 模块划分

### 2.1 功能模块

| 模块 | 前缀 | 描述 | 主要功能 |
|------|------|------|----------|
| **认证管理** | `AuthController` | 身份认证与用户管理 | 登录、注册、用户信息
| **问答与AI** | `QaController` | 智能问答与互动 | 发起提问、提交AI反馈 |
| **知识图谱** | `GraphController` | 知识网络管理 | 获取全量图谱、获取节点详情 |
| **成长记录** | `RecordController` | 个人记录与标签管理 | 记录生命周期管理 |
| **数据看板** | `DashboardController` | 仪表盘统计 | 获取多维度的统计数据 |
| **时间轴** | `TimelineController` | 历史回顾 | 串联并展示成长时间线 |
| **系统元数据** | `MetaController` | 系统核心元数据 | 基础配置与系统标签管理 |

### 2.2 数据模型层次
```
Data Transfer Objects (DTO)
├── Auth
│   └── 基础 Map 响应结构
├── RecordRequest (记录输入模型)
├── QA
│   ├── AskRequest (提问输入模型)
│   └── FeedbackRequest (反馈输入模型)
└── Graph
    ├── KnowledgeGraphDTO (完整图谱输出)
    │   ├── GraphNodeDTO
    │   └── GraphEdgeDTO
    └── KnowledgeNodeDetailDTO (节点详情输出)
```

##  认证管理 API

### 3.1 用户注册

**接口信息**：
- **方法**：`POST`
- **路径**：`/api/auth/register`
- **描述**：注册新用户

**请求参数**：
```json
{
    "username": "developer_01",
    "password": "securepassword123",
    "nickname": "LogicMaster"
}
```

**响应示例**：
```json
{
    "userId": 1,
    "username": "developer_01",
    "nickname": "LogicMaster",
    "message": "注册成功"
}
```

### 3.2 用户登录

**接口信息**：
- **方法**：`POST`
- **路径**：`/api/auth/login    `
- **描述**：获取 JWT 认证 Token。


**请求参数**：

```json
{
    "username": "developer_01",
    "password": "securepassword123"
}
```

**响应示例**：

```json
{
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI...",
    "username": "developer_01",
    "nickname": "LogicMaster"
}
```

### 3.3 用户登出

**接口信息**：
- **方法**：`POST`
- **路径**：`/api/auth/logout`
- **描述**：统一前后端退出语义，前端需主动清理本地 Token。


**响应示例**：
```json
{
    "ok": true
}
```

**接口信息**：
- **方法**：`GET`
- **路径**：`/api/auth/me`
- **描述**：解析 Token 返回当前登录人信息。

**响应示例**：

## 安全与认证

### 9.1 认证机制

**Token传递**：
客户端在完成 `/api/auth/login` 接口获取 Token 后，需要在后续所有受保护 API 的请求头中携带：
Authorization: Bearer <Your_Token>

**上下文管理**：
后端拦截器解析 Token 后，通过 `UserContext.userId()` 进行数据级隔离，确保用户只能访问与修改属于自己的记录与图谱数据。

## API使用示例

### 11.1 Python 客户端示例

```python
import requests

class CoBloomClient:
    def __init__(self, base_url="http://127.0.0.1:8080"):
        self.base_url = base_url
        self.token = None
        
    def login(self, username, password):
        res = requests.post(f"{self.base_url}/api/auth/login", json={
            "username": username,
            "password": password
        })
        res.raise_for_status()
        self.token = res.json().get("token")
        
    def _get_headers(self):
        return {"Authorization": f"Bearer {self.token}"} if self.token else {}

    def create_note(self, title, content, tags=None):
        res = requests.post(
            f"{self.base_url}/api/records",
            headers=self._get_headers(),
            json={
                "title": title,
                "content": content,
                "tags": tags or []
            }
        )
        return res.json()

    def ask_ai(self, question):
        res = requests.post(
            f"{self.base_url}/api/qa/ask",
            headers=self._get_headers(),
            json={"question": question}
        )
        return res.json()

# 使用流程
if __name__ == "__main__":
    client = CoBloomClient()
    client.login("developer_01", "securepassword123")
    
    # 录入一条专业笔记
    client.create_note(
        title="复习最大项", 
        content="在最大项 $M_k$ 逻辑中，原变量对应0，反变量对应1。", 
        tags=["数字逻辑"]
    )
    
    # 向系统发起提问
    ai_response = client.ask_ai("最大项的变量规则是什么？")
    print(ai_response['answer'])
```

## 附录

### A.1 HTTP状态码说明

| 状态码 | 描述 | 使用场景 |
|--------|------|----------|
| 200 | OK | GET, POST, PUT, DELETE 请求成功处理 |
| 401 | Unauthorized | Token 缺失、失效或格式错误 |
| 403 | Forbidden | 权限不足 |
| 404 | Not Found | 资源不存在 |
| 500 | Internal Error | 服务器错误 |
