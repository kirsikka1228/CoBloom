## 1. 项目概述

### 1.1 项目名称

**AI 原生个人知识库与智能问答平台**

### 1.2 项目类型

Web 方向课程设计项目。

### 1.3 项目背景

在学习、科研和开发过程中，用户通常会积累大量 Markdown 笔记，例如课程笔记、技术文档、项目记录、论文阅读记录等。传统笔记系统虽然可以完成内容存储和分类管理，但在知识检索、知识关联发现、内容摘要和智能问答方面能力较弱。

本项目的定位不是简单的 Markdown 管理系统，而是构建一个 AI
驱动的个人知识库系统。系统参考 AI Knowledge Base 思想：

- 用户输入原始知识资料；
- AI自动理解、整理和结构化知识；
 - 构建知识关系网络； 
- 基于知识库进行智能问答； 
- 用户探索过程产生的新知识反哺知识库。

核心理念：

> Markdown 是知识输入形式，而不是最终知识形态。

### 1.4 项目目标

本系统的主要目标包括：

1. 支持用户注册、登录和个人知识库管理；
2. 支持 Markdown 笔记的新建、编辑、删除、查看和检索；
3. 支持单个 `.md` 文件上传并解析为笔记；
4. AI 自动知识编译（Knowledge Compiler）；
5. 自动摘要、关键词、概念和实体提取；
6. 文档切片与向量索引；
7. 支持基于用户个人知识库的 RAG 智能问答；
8. 支持问答结果展示引用来源，便于用户追溯答案依据；
9. 支持根据标签、关键词和内容相似度推荐相关笔记；
10. 支持以图谱形式展示笔记、标签、关键词和相关笔记之间的关系；
11. 支持问答历史记录保存和查看；
12. 支持 Docker + Nginx 部署。

### 1.5 系统总体架构

```
Markdown / 文件输入
        ↓
Knowledge Compiler
        ↓
结构化知识层
        ↓
--------------------------------
|              |               |
摘要         知识图谱        向量索引
--------------------------------
        ↓
AI Knowledge Engine
        ↓
问答 / 推荐 / 知识探索
```

## 2. 项目范围

### 2.1 本期实现范围

本项目本期主要实现一个面向个人用户的 AI 知识库平台。系统支持多用户注册登录，但每个用户只管理自己的个人知识库，不涉及团队协作。

本期核心功能包括：

| 模块              | 功能范围                                   |
| ----------------- | ------------------------------------------ |
| 用户认证模块      | 注册、登录、退出、JWT 鉴权                 |
| 笔记管理模块      | Markdown 笔记增删改查、搜索、标签、分类    |
| Markdown 编辑模块 | 在线编辑、实时预览、保存                   |
| 文件上传模块      | 上传单个 `.md` 文件并生成笔记              |
| AI 摘要模块       | 对笔记生成摘要和关键词                     |
| RAG 问答模块      | 基于个人知识库检索后生成回答               |
| 引用来源模块      | 展示 AI 回答所依据的笔记片段               |
| 问答历史模块      | 保存和查看历史问答                         |
| 关联推荐模块      | 根据标签、关键词和相似度推荐相关笔记       |
| 知识图谱模块      | 展示笔记、标签、关键词、相关笔记之间的关系 |
| 部署模块          | Docker 部署后端和数据库，Nginx 部署前端    |

### 2.2 本期不实现范围

为保证 12 天内完成较高质量的课程设计，本期明确不实现以下功能：

| 功能                   | 不实现原因                         |
| ---------------------- | ---------------------------------- |
| 团队知识库协作         | 权限设计复杂，开发周期不足         |
| 多人实时协同编辑       | 技术复杂度高，不属于核心需求       |
| PDF / Word 文件解析    | 文件解析成本较高，容易影响核心功能 |
| 图片附件管理           | 与 RAG 核心功能关系不大            |
| Neo4j 知识图谱         | 架构较重，不适合短周期课程设计     |
| Milvus / Chroma 向量库 | 额外部署和学习成本较高             |
| 联网搜索问答           | 容易偏离个人知识库问答定位         |
| 复杂管理员后台         | 课程设计收益较低                   |
| 移动端适配             | 本项目优先完成 PC Web 端           |
| 第三方登录             | 非核心功能                         |
| 找回密码 / 邮箱验证码  | 非核心功能                         |

## 3. 用户角色与权限

### 3.1 用户角色

本系统初版设计两个角色，但主要实现普通用户功能。

| 角色     | 说明                                           | 本期实现程度 |
| -------- | ---------------------------------------------- | ------------ |
| 普通用户 | 注册登录后管理自己的个人知识库、笔记和问答记录 | 必做         |
| 管理员   | 用于系统配置、用户管理等扩展功能               | 可选 / 弱化  |

### 3.2 普通用户权限

普通用户可以：

1. 注册账号；
2. 登录系统；
3. 创建、编辑、删除自己的 Markdown 笔记；
4. 上传单个 `.md` 文件生成笔记；
5. 查看自己的笔记列表和笔记详情；
6. 为笔记添加标签和分类；
7. 对自己的笔记生成 AI 摘要和关键词；
8. 基于自己的知识库进行 AI 问答；
9. 查看 AI 回答引用来源；
10. 查看自己的问答历史；
11. 查看自己的知识图谱；
12. 查看系统推荐的相关笔记。

普通用户不可以：

1. 访问其他用户的笔记；
2. 查看其他用户的问答记录；
3. 修改其他用户的数据；
4. 直接查看或修改系统大模型 API Key。

### 4.6 RAG 智能问答模块

#### 4.6.1 功能说明

RAG 智能问答是本项目核心功能。用户提出问题后，系统从用户个人知识库中检索相关笔记片段，将检索结果作为上下文传入大语言模型，生成基于知识库内容的回答。

#### 4.6.2 功能列表

| 功能         | 描述                             | 优先级 |
| ------------ | -------------------------------- | ------ |
| 输入问题     | 用户在 AI 问答页面输入问题       | 必做   |
| 问题向量化   | 将用户问题转换为向量             | 必做   |
| 知识库检索   | 从当前用户笔记分块中检索相关内容 | 必做   |
| Top-K 召回   | 返回最相关的 3～5 个片段         | 必做   |
| AI 生成回答  | 将检索片段作为上下文生成答案     | 必做   |
| 引用来源展示 | 展示答案依据的笔记标题和片段     | 必做   |
| 保存问答记录 | 保存用户问题、AI 回答和引用来源  | 必做   |
| 查看历史问答 | 用户查看历史问答记录             | 建议做 |

#### 4.6.3 业务流程

RAG 问答流程如下：

1. 用户输入问题；
2. 后端接收问题并识别当前用户；
3. 调用 embedding 模型将问题向量化；
4. 从当前用户的笔记分块向量表中检索相似片段；
5. 选取 Top-K 个相关片段；
6. 将问题和相关片段拼接为 Prompt；
7. 调用大语言模型生成回答；
8. 返回 AI 回答和引用来源；
9. 保存问答记录。
------

### 4.7 文档分块与向量检索模块

#### 4.7.1 功能说明

该模块负责将 Markdown 笔记拆分为较小文本片段，并为每个片段生成向量，用于 RAG 检索和相关笔记推荐。

#### 4.7.2 功能列表

| 功能       | 描述                             | 优先级 |
| ---------- | -------------------------------- | ------ |
| 文本分块   | 将 Markdown 正文切分为多个文本块 | 必做   |
| 向量生成   | 调用 embedding API 生成文本向量  | 必做   |
| 向量存储   | 将文本块和向量保存到数据库       | 必做   |
| 问题向量化 | 将用户问题转换为向量             | 必做   |
| 相似度计算 | 计算问题向量与文本块向量相似度   | 必做   |
| Top-K 检索 | 返回最相关的若干文本块           | 必做   |

#### 4.7.3 分块规则

建议采用较简单稳定的分块策略：

1. 优先按 Markdown 标题和段落分块；
2. 如果某段文本过长，则按字数继续切分；
3. 每个文本块建议控制在 500～800 字；
4. 相邻文本块可保留 50～100 字重叠内容；
5. 每个文本块需要记录来源笔记 ID 和位置序号。


### 4.8 关联推荐模块

#### 4.8.1 功能说明

关联推荐模块用于在用户查看某篇笔记时，推荐知识库中与当前笔记主题相近的其他笔记。

#### 4.8.2 功能列表

| 功能           | 描述                       | 优先级 |
| -------------- | -------------------------- | ------ |
| 相关笔记推荐   | 在笔记详情页展示相关笔记   | 必做   |
| 标签匹配       | 根据相同标签推荐笔记       | 必做   |
| 关键词匹配     | 根据 AI 关键词推荐笔记     | 必做   |
| 向量相似度推荐 | 根据笔记内容向量相似度推荐 | 建议做 |
| 推荐跳转       | 点击推荐笔记进入详情页     | 必做   |

#### 4.8.3 推荐规则

推荐结果综合考虑：

1. 相同标签数量；
2. 相同关键词数量；
3. 笔记内容向量相似度；
4. 最近更新时间。

推荐分数可以初步设计为：

```text
推荐分数 = 相同标签数 × 2 + 相同关键词数 × 1 + 向量相似度 × 5
```

系统按照推荐分数降序排列，返回前 5 篇相关笔记。

## 8. 非功能需求

### 8.1 安全性需求

1. 用户密码必须加密存储；
2. 系统采用 JWT 进行接口认证；
3. 后端接口需要校验用户身份；
4. 用户数据必须隔离；
5. 大模型 API Key 不能暴露在前端；
6. 文件上传需要限制文件类型和大小；
7. 删除和修改操作需要校验数据归属。

### 8.2 性能需求

1. 普通笔记列表查询响应时间应在可接受范围内；
2. AI 摘要和问答允许较长响应时间，但前端需要显示加载状态；
3. RAG 检索默认返回 Top 5 片段；
4. 图谱节点过多时应限制展示数量，避免页面卡顿；
5. 本课程设计项目主要面向小规模数据，不以大规模并发为目标。

### 8.3 可用性需求

1. 页面布局清晰，核心操作入口明显；
2. AI 生成过程需要有加载提示；
3. 操作失败时需要显示错误提示；
4. 删除操作需要二次确认；
5. 引用来源可点击跳转原笔记；
6. Markdown 编辑和预览应具备较好的可读性。

### 8.4 可维护性需求

1. 前后端分离；
2. 后端采用分层结构：Controller、Service、Mapper、Entity；
3. 后端接口使用 Swagger / Knife4j 生成接口文档；
4. 前端按页面和模块划分目录；
5. AI 调用服务单独封装，便于替换模型；
6. RAG 检索逻辑单独封装，便于扩展。

### 8.5 部署需求

1. 后端使用 Spring Boot 打包部署；
2. 前端使用 Vue3 打包后通过 Nginx 部署；
3. 数据库使用 MySQL 或 PostgreSQL；
4. 推荐使用 Docker Compose 管理服务；
5. 配置文件中应支持设置数据库连接、大模型 API Key 等参数。

## 10. 核心业务流程

### 10.1 用户使用流程

```text
注册 / 登录
    ↓
进入首页仪表盘
    ↓
新建或上传 Markdown 笔记
    ↓
编辑并保存笔记
    ↓
生成 AI 摘要和关键词
    ↓
系统对笔记分块并向量化
    ↓
用户进行知识库问答
    ↓
系统检索相关片段并生成回答
    ↓
展示回答和引用来源
    ↓
用户查看相关笔记推荐和知识图谱
```

### 10.2 笔记保存与向量更新流程

```text
用户保存笔记
    ↓
后端保存 Markdown 原文
    ↓
清理该笔记旧分块和旧向量
    ↓
按照标题 / 段落进行文本分块
    ↓
调用 embedding API 生成向量
    ↓
保存分块文本和向量
    ↓
更新笔记状态
```

### 10.3 AI 问答流程

```text
用户输入问题
    ↓
后端生成问题向量
    ↓
从当前用户知识库中检索 Top-K 相关片段
    ↓
拼接 Prompt
    ↓
调用大语言模型
    ↓
生成回答
    ↓
返回回答和引用来源
    ↓
保存问答历史
```

### 10.4 知识图谱生成流程

```text
读取当前用户笔记
    ↓
读取笔记标签
    ↓
读取 AI 关键词
    ↓
计算或读取相关笔记关系
    ↓
生成节点和边
    ↓
前端使用 ECharts Graph 展示
```

系统应满足：

1. 笔记详情页可以展示相关笔记推荐；
2. 推荐依据包括标签、关键词或内容相似度；
3. 知识图谱页可以展示笔记、标签、关键词之间的关系；
4. 点击笔记节点可以跳转笔记详情。

### 12.4 部署验收

系统应满足：

1. 前端可以正常打包；
2. 后端可以正常启动；
3. 数据库连接正常；
4. 可通过浏览器访问系统；
5. Docker / Nginx 部署方案可说明或演示。

## 13. 风险分析与应对

| 风险              | 说明                       | 应对方案                           |
| ----------------- | -------------------------- | ---------------------------------- |
| AI API 调用失败   | 网络、额度、Key 配置问题   | 做异常处理，准备备用模型或模拟数据 |
| pgvector 配置困难 | 环境配置可能耗时           | 准备 MySQL/JSON 向量存储备选方案   |
| RAG 效果不稳定    | 检索结果可能不准确         | 控制测试数据，优化分块和 Prompt    |
| 功能范围过大      | 12 天周期有限              | 优先完成 P0，再做 P1               |
| 知识图谱过于复杂  | 真正图谱开发成本高         | 做轻量关系图，不引入 Neo4j         |
| 前后端联调耗时    | 接口字段不一致             | 第 2 天先确定接口文档              |
| UI 页面过多       | 三人开发压力大             | 保留 9 个核心页面，弱化管理员功能  |
| 部署问题          | Docker、Nginx 配置可能出错 | 最迟第 11 天开始部署，不拖到最后   |

## 14. 初版功能边界总结

本项目最终初版功能边界确定如下：

| 模块          | 初版选择                                                     |
| ------------- | ------------------------------------------------------------ |
| 用户范围      | 多用户个人知识库                                             |
| 协作方式      | 不支持团队协作                                               |
| Markdown 来源 | 在线编辑 + 单个 `.md` 上传                                   |
| AI 问答范围   | 基于个人知识库的 RAG 问答                                    |
| 引用来源      | 必须展示                                                     |
| RAG 技术      | 文本分块 + embedding + Top-K 检索 + 大模型生成               |
| 向量存储      | 优先 PostgreSQL + pgvector，备选普通数据库存储               |
| 知识图谱      | 轻量级知识关系图                                             |
| 图谱节点      | 笔记、标签、关键词                                           |
| 图谱边        | 笔记-标签、笔记-关键词、笔记-相关笔记                        |
| 关联推荐      | 标签 + 关键词 + 向量相似度                                   |
| 权限安全      | Spring Security + JWT + BCrypt + 用户数据隔离                |
| 页面范围      | 登录、注册、首页、笔记列表、编辑、详情、AI 问答、知识图谱、问答历史 |
| 部署方式      | Docker + Nginx                                               |
| 不做内容      | 团队协作、PDF/Word、Neo4j、复杂管理员、联网搜索              |
